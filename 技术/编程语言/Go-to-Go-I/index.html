Go to Go:I
发表于2020-10-13|更新于2020-10-13|技术编程语言
|字数总计:812|阅读时长:2分钟|阅读量:

概述

Go是一种开源的程序设计语言，使得人们能够方便的构建简单、可靠、高效的软件。

Go于2009年11月发布，发明人是Robert Griesemer、Rob Pike和Ken Thompson，Go可视为C的一种升级版本，同时基于多种其他语言，实现了高效的并发、灵活的数据抽象和面向对象、自动化的内存管理：垃圾回收。

  • Go继承了C的表达式语法、控制流语句、基本数据类型、按值调用的形参传递和指针，以及编译成高效机器码的核心要点
  • Go继承了Pasca、Modula-2、Oberon、Oberon-2的包概念，包、导入、声明的语法
  • Go继承了CSP（Communicating Sequential Process，通信顺序进程）的基本思想，在CSP中，程序就是一组无共享状态进程的并行组合，进程间的通信和同步采用通道完成

入门

基础

Go是编译型语言：Go的编译器将源文件转变为机器相关的原生二进制指令（机器码），命令是：

  • go run xxx.go编译并运行程序，一次性
  • go build xxx.go编译程序，生成可执行文件

Go使用包组织代码，包类似库/模块，由一个或多个go源文件组成，具有一定的功能，每一个源文件的开始都使用package声明，然后紧跟它导入的其他包的列表，然后是程序声明；main包用于定义独立的可执行程序，而不是库，main包中main函数作为程序执行的入口，import导入程序所依赖的包，必须跟在package之后；导入包必须精确，在缺少导入或者==存在不需要的包==时，编译失败；import可每次导入一个文件，也可导入由圆括号包起来的包列表。

Go无需分号作为语句结尾，除非多个语句或者声明出现在同一行。注意：跟在特定符号之后的换行符被转换为分号，由此换行会影响对Go代码的解析，如：

  • 关键字func必须与{符号在同一行
  • x+y表达式中，换行不能放到+的前面

Go对代码的格式化要求十分严格，可使用gofmt工具格式化指定包内的所有文件或者当前文件夹的文件

命令行参数

Go中命令行参数以os.Args进行访问。

  • os.Args是一个字符串slice：一个动态容量的顺序数组
  • slice可通过s[i]访问单个元素，可通过s[m:n]访问一段连续子区间，前闭后开，类比于Python
  • 通过len(s)得到s的长度

一些要点：

  • var关键字声明变量；:=用于短变量声明

  • i++是自增语句，而非表达式，故无返回值，因此j=i++不合法；且只有后缀自增、自减语句

  • for循环无需小括号、必需大括号、左大括号必须与后置（更新）语句同一行

  • range关键词用于迭代，每次产生一对值：索引和索引处元素的值

  • ==Go不允许存在无用的临时变量，可使用空标识符-（下划线）代替==
