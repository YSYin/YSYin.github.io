Go to Go: III
发表于2020-10-14|更新于2020-10-16|技术编程语言
|字数总计:1.1k|阅读时长:3分钟|阅读量:

基本数据

Go的数据类型分为四大类

  • 基础类型：数值、字符串、布尔型
  • 聚合类型：数组、结构体
  • 引用类型：指针、slice、map、函数、通道等
  • 接口类型

整数

Go具备有符号整数和无符号整数

  • 有符号整数：int8(8位)、int16(16位)、int32(32位)、int64(64位)
  • 无符号整数：uint8(8位)、uint16(16位)、uint32(32位)、uint64(64位)
  • int和uint的大小与特定的平台和编译器相关
  • rune类型是int32类型的同义词，指明一个值是Unicode码点
  • byte类型是uint8类型的同义词，指明一个值是原始数据
  • uintptr是足以完整存放指针的无符号整数，仅仅用于底层编程
  • 不同类型之间必须要要显式类型转换（包括int和int32）

Go的二元操作符及优先级

  • 最高优先级：*、/、%、<<、>>、&、&^
  • 次高优先级：+、-、|、^
  • 中等优先级：==、!=、<、<=、>、>=
  • 次低优先级：&&
  • 最低优先级：||

取模运算符%仅能用于整数，取模余数的正负号总是与被除数一致。

整数相除，商也是整数

溢出会舍弃溢出的高位，有符号类型可能形成负值

^作为二元运算符时为异或操作，作为一元运算符时为取反操作

&^为按位清除：第二操作数某位为1，则将第一操作数的对应位清除为0

移位运算中，位移长度必须为无符号型，被操作数可以是有符号型也可以是无符号型

无符号数右移以0补左边空位，有符号数右移按符号位的值填补空位

无符号整数往往只用于位运算符和特定算术运算符，一般而言，无符号整数极少用于表示非负值（如数组长度等等）

通常，将某种类型的值转换为另外一种，必须显式转换

相同位数的整型转换不会引起值位的变化，仅告知编译器如何解读该值；缩减大小的整型转换以及整型与浮点型之间的相互转换，可能会改变值或损失精度

  • 浮点型转成整型，会舍弃小数部分，趋零截尾（正值向下取整、负值向上取整）
  • 转换操作数的值超出目标类型的取值范围时，结果依赖于具体实现

fmt输出数字时

  • %d以十进制表示输出、%x以十六进制小写表示输出、%o以八进制表示输出
  • %后的修饰[1]表示重复使用第一个操作数，修饰#表示输出前缀0/0x/0X

字符字面值

  • 字符写在单引号中
  • %n输出码值，%c输出字符，%q输出带有单引号的字符

浮点数

Go中有两种大小的浮点数float32和float64，算术特性遵从IEEE 754标准

math.MaxFloat32是float32的最大值，大约为3.4e38；最小值约为1.4e-45；math.MaxFloat64是float64的最大值，大约为1.8e308；最小值约为4.9e-324

十进制下，float32的有效数字大约是6位，能精确表示的正整数范围有限，float64的有效数字大约是15位；绝大多数情况下，应该选择float64

Printf的输出谓词：

  • %g，自动保持足够的精度、并选择最简洁的表示方式输出浮点数
  • %e，有指数的输出方法
  • %f，无指数的输出方法
  • 三个谓词都能掌控输出宽度和数值精度

math包具有函数创建和判断：

  • 正无穷大和负无穷大，表示超出最大许可值的数以及除以0的商
  • NaN，表示数学上无意义的运算结果（0/0，sqrt(-1)）
    • math.isNaN判断是否为NaN
      • x != x

复数

Go中有两种大小的复数complex64和complex128，二者分别由float32和float64组成，内置的complex()函数根据指定的实部和虚部创建复数，内置的real()和image()函数分别提取复数的实部和虚部

在源码中，在浮点数或十进制整数后紧跟字母i，则变成一个虚数

math/cmplx提供了复数运算所需的库函数

布尔值

只有两个值：true和false
