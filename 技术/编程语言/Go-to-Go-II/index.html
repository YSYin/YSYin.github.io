程序结构

Go语言，包括其他编程语言的程序架构：==变量存储值；简单表达式通过加或减等操作合并成大的；基本类型通过数组和结构体进行聚合；表达式通过if和for等控制语句决定执行顺序；语句被组织成函数用于隔离和复用；函数被组织成源文件和包==。

名称

Go中实体：函数、变量、常量、类型、语句标签和包

  • 命名规则：

    • 开头是一个字母（Unicode字符）或下划线
    • 后面是任意数量的字符、数字和下划线
    • 区分大小写
    • 25个关键字不能作为名称
    • 若干预声明常量、类型和函数可在声明中使用它，但有冲突的风险
    • 包名总是由小写字母组成

  • 有效域：

    • 函数内声明的实体只在函数局部有效
    • 函数外声明的实体对包内全部源文件有效
      • 实体名称以大写字母开头，是导出的，对包外文件可见、可访问

  • 命名风格：

    • 驼峰式优先
    • 首字母缩写词同大同小，如HTML/html、ASCII/ascii

声明

声明给一个程序实体命名，并且设定其部分或全部属性，有四种主要声明

  • var变量
  • const常量
  • type类型
  • func函数

变量

var声明创建一个具体类型的变量，然后附加一个名称，设置它的初始值

  • 基本形式：var name type = expression

  • 类型type和表达式expression部分可以省略一个，但不能都省略：

    • 如果类型省略，变量类型由表达式决定
    • 表达式省略，初始值为对应类型的零值（0 false ""nil）
    • 由此，Go中不存在未初始化变量

  • 可以声明一个变量列表，并使用对应的表达式列表对其初始化，忽略类型允许声明多个不同类型的变量

    • var i, j, k int // int int int
var b, f, s = false, 2.2, "ff" // bool float64 string

    • 按照b-c-a的顺序初始化

  • 如果包由多个文件组成，go工具会在调用编译器前将文件排序，然后编译器按照收到文件的顺序进行初始化

  • 包级别的变量，生命周期从被初始化开始

  • 任何文件可包含任意数量的init函数：func init(){}，该函数不能被调用和引用，在程序启动的时候，init函数按照声明的顺序自动执行

  • 包的初始化按照在程序中导入的顺序进行，依赖顺序进行，每次初始化一个包，main包最后初始化

作用域

作用域：可以用到声明时所声明名字的源代码段，编译时属性

生命周期：在程序执行期间能够被程序的其他部分所引用的起止时间，运行时属性

语法块：大括号围起来的语句序列

词法块：语法块，以及没有显示包含在大括号里的声明代码

全局块：包含了全部源代码的词法块

声明的词法块决定声明的作用域大小

  • int、len等内置类型、函数或常量在全局块中声明并对整个程序可见

  • 包级别的声明对整个包的文件可见

  • 导入的包是文件级别的，对同一个文件内可见，对同一个包的不同文件不可见

  • 控制流标签（break、continue、goto）的作用域是整个外层的函数

  • 一个程序可包含多个同名的声明，前提是在不同词法块中

  • 编译器遇到一个名字的引用时，将从最内层的封闭词法块到全局块寻找其声明

    • 没有找到，报undeclared name错误
    • 内层、外层词法块都存在，内层声明覆盖外层声明

  • 短变量声明依赖一个明确的作用域，即如果本层语法块未出现显式声明，则短声明直接声明变量为局部变量，不会考虑上层语法块是否存在该变量的声明
